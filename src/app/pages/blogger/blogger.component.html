How It Works

Overview

This project is an update from an older database college class project. At first glance one can notice this site looks very similar to Twitter. However, there were two goals in mind while updating this project:

  • Being able to develop any UI design at sight without worrying about originality and making it responsive
  • Being able to communicate front and back between the client and the backened using CRUD and RESTful API requests to fetch data or make any updates to it

Aside from all these points, the first thing a user will encounter is the login page. A new user has the option to register as well as mentioned below under the login submit button. Any time a user visits one of the existing endpoints without signing in first will always be returned here or when their user session expires which is meant to last up to 24 hours before it asks users to log back in. Once the user signs in, they will be redirected to the homepage where the first thing that will show up will be blogs from people the user follows. A user can also choose to log out any time by clicking their username over the bottom left of the page.

Posting

If you used Twitter before then posting will be very similar with Blogger. The main difference is that this site requires users to tag their blogs with hobbies.


In this case, a user will be forced to select hobbies of their choice first before attempting to post any blogs. The reason for this was so users can have the ability to filter out blogs they'd like to see or not based on their tags. It also gives users an idea of what type of people they want to follow or not. This is quite an issue with a lot of social media platforms where people end up arguing when certain topics are touched upon throughout posts. Therefore, the site gives a solution to this problem by filtering out the type of blogs a user wishes to see to avoid the need of unfollowing people anytime a user reads any unpleasant posts from someone else. More information about filtering can be found in one of the sections below.

Searching

There are two methods for searching users. One of these methods is by knowing their username. The search field will help narrow down similar users based on the characters or substring that a user is typing while searching. The other method is by selecting the search option for users with mutual hobbies. This method makes it easier for users to connect with new people and especially with those who share the same scope of interests. In the case a user follows another user who already follows them then they will start showing up as friends on the right.

Profile

This is where a user can find all of their own personal blogs. These blogs don't show up with the rest of the other blogs from the homepage but this can always be changed. Some might want to see their blogs with rest over the home page while others may not. Moreover, a user can find more information about someone such as their total number of blogs, followers, and users they follow. If the user clicks over the followers or followings, then they will be redirected to another page where it displays all of those users respestively.

Filtering

Going back to the homepage, a message appears on top asking what you would like to check. This is actually a button, and as a result will display options for filtering the types of blogs a user wishes to see. All of these options are subject to change. However, this is just a sample of possible options that could benefit users to use.

Commenting

A user can select any blog they wish to comment while also giving a positive or negative rating attached it. However, a user is limited to commenting a blog only once at the moment. They also cannot comment their own blogs either. Certainly the limit can always be changed. Yet, the reason it was developed this way was to practice database queries that implements one-to-one, one-to-many, and many-to-many database relationships. Afterall, this project was initially assigned from a database class.
